# Daily Coffee 發文 SOP（給同事）

> 這份給所有要用後台發文章的同事看。**不用懂程式碼，照步驟走就好**。

---

## 📌 我可以用這個工具做什麼？

不用寫任何 HTML，就能發一篇結構完整的文章到 https://dailycoffee.matrix.com.tw/ ，包含：

- 自動產生 SEO meta 標籤（Google 搜尋會用）
- 自動產生社群分享卡片（LINE / FB / Threads 轉貼會漂亮）
- 自動更新首頁文章列表、網站地圖
- 支援文末掛商品卡（matrix.com.tw 的商品）

**整條流程：填表 → 預覽 → 按發佈 → 30 秒後上線**

---

## 🔗 工具網址

打開這個網址：

```
https://dailycoffee.matrix.com.tw/admin/
```

建議加到瀏覽器書籤。

## 🔑 第一次使用：輸入發佈密碼

第一次按「🚀 發佈文章」時，會彈窗問密碼。向主管要 **ADMIN_TOKEN** 貼進去即可。

密碼會自動記在瀏覽器裡（localStorage），之後不會再問。

**換電腦或換瀏覽器會再問一次**，所以請把 token 存在安全的地方。

> 萬一打錯了，瀏覽器開 Console（Mac 按 ⌘+⌥+J），貼 `resetAdminToken()` 按 Enter，重整頁面再試。

## 📝 發文步驟

### 步驟一：基本資訊

| 欄位 | 怎麼填 |
|---|---|
| **標題** | 60 字以內，寫給讀者看的中文標題 |
| **Slug**（網址後綴） | 按「✨ 從標題自動產生」即可。純中文標題會呼叫 AI 翻譯成英文，約 3 秒 |
| **分類** | 可多選，至少選一個。目前 6 種：咖啡知識 / 咖啡地圖 / 新聞動態 / 活動展覽 / KOL 專訪 / CBTI 人格 |
| **摘要** | 80–120 字。也可以留空，寫完內文後按「✨ 從內文自動產生」 |
| **標籤** | 逗號分隔，建議 3–6 個，例如「手沖,V60,粉水比」 |

---

### 步驟二：封面圖

**兩種方式二選一：**

**📁 上傳圖片**
- 點框框選擇檔案，≤ 1MB，jpg/png/webp 都可
- 系統會自動裁切成 1200×630 + 轉成 webp（不用自己處理尺寸）

**🎨 AI 產圖**
- 切到 AI 分頁，輸入英文 prompt（中文也可但效果稍差）
- 範例 prompt：`minimal flat-lay photo of V60 pour over dripper with fresh coffee beans, soft morning light, editorial magazine style`
- 按鈕 → 等 5–15 秒 → 自動出圖 + 裁切完成

---

### 步驟三：寫內文（三選一，看你的情境）

#### ✍️ 方式 A：富文字編輯器（最常用）

像 Word 一樣打字就好。工具列有：
- **標題（h2 / h3）**
- **粗體 / 斜體**
- **連結**
- **清單（有序 / 無序）**
- **引用 blockquote**
- **🖼️ 插入圖片**（點工具列最後面那顆）
- **清除格式**

**中段插入圖片：** 點工具列的 🖼️ 按鈕選檔案，會直接嵌在文章段落裡。系統會在發佈時自動存成本地檔。

---

#### 📋 方式 B：貼 AI 草稿

如果你在 Claude / ChatGPT 聊天視窗已經有一篇寫好的文章：

1. 切到「📋 貼 AI 草稿」分頁
2. 整段複製貼上（支援 Markdown 或 HTML）
3. 按「轉換並載入富文字」
4. 系統會自動：
   - 剝除奇怪的 script / style / class
   - h1 降級成 h2
   - 把 Markdown 的圖片語法 `![](...)` 保留
5. 自動跳到富文字分頁讓你再微調

---

#### </> 方式 C：HTML 原始碼（進階）

只有在你要發**像 matrix.com.tw 那種雜誌版型**（含 info-card、pull-quote、計算器…）才用這個。

- 把完整 HTML 貼進去
- 支援 `<section>` / `<div class="...">` / `<table>` 等複雜結構
- 送出時仍會擋掉 `<script>` / `<iframe>` / `onclick`

**如何在 HTML 中插入圖片：**
1. 點工具列「🖼️ 上傳圖片取得路徑」
2. 選檔案、填 alt 文字
3. 複製系統產生的 `<img src="...">` 片段
4. 貼到 HTML 你想放的位置

---

### 步驟四：文末商品區（選填）

想在文末掛商品卡（像 [這篇](https://www.shinygoods.tw/hitachi-seaside-park-nemophila-2026) 的風格）：

1. 點「＋ 新增商品」
2. 貼 matrix.com.tw 的商品網址
3. 按「🔍 自動抓取商品資料」→ 系統抓商品圖和名稱
4. **售價要手動填**（1shop 的價格是動態的，無法自動抓）
5. 可選填原價（會顯示刪除線優惠效果）
6. 按「加入商品」

每篇最多建議 10 張卡。想移除按卡片右上角的 ×。

---

### 步驟五：預覽 + 發佈

1. 按 **「👁️ 預覽文章」** → 彈出全螢幕預覽，看起來跟上線後一模一樣
2. 預覽沒問題後按 **「🚀 發佈文章」**
3. 跳確認框 → 按 OK
4. 等 15–30 秒（上傳、裁切、commit、部署）
5. 成功後會給你文章網址，點進去驗收

## ⚠️ 常見錯誤與解法

| 訊息 | 原因 | 怎麼辦 |
|---|---|---|
| 「401 未授權」 | ADMIN_TOKEN 打錯 | Console 跑 `resetAdminToken()`，重試 |
| 「409 slug 已存在」 | 跟既有文章撞名 | 改標題或手動改 slug |
| 「slug 格式錯誤」 | slug 有空白或中文 | 按「✨ 從標題自動產生」讓 AI 翻譯 |
| 「至少選一個分類」 | 分類沒勾 | 回上方勾分類 |
| 「摘要不可超過 120 字」 | 摘要太長 | 縮短或按自動產生重新產 |
| 「內文過短」 | 少於 50 字 | 寫多一點 |
| 「檔案超過 1MB」 | 封面圖太大 | 壓縮後再傳，或改用 AI 產圖 |
| 「MiniMax 回應 XXX」 | AI 產圖失敗 | 換 prompt 再試；持續失敗通知主管 |
| 「AI 翻譯失敗」 | OpenAI 服務異常 | 手動打英文 slug，或稍後再試 |
| 發佈後網址打開是 404 | Vercel 還在部署 | 等 30 秒重整。超過 2 分鐘通知主管 |

## 📋 發文前檢查清單（建議自己跑過一遍）

- [ ] 標題 ≤ 60 字、有吸引力
- [ ] Slug 是英文小寫 + 連字號（自動產生就 OK）
- [ ] 勾了至少一個分類
- [ ] 摘要 80–120 字（不要空或太短，影響 SEO）
- [ ] 封面圖清楚、有設計感
- [ ] 內文至少 300 字，有 2 個以上 h2 小標
- [ ] 圖片都有 alt 描述（富文字編輯器會提醒）
- [ ] 外部連結加入正確的品牌名
- [ ] 商品卡（若有）：售價、連結都對
- [ ] 按「👁️ 預覽文章」檢查一次
